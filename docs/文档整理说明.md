## 六、文档维护原则

1. **以当时修改为准**：代码与文档不一致时，以当时修改后的版本为准；修改时**保留更改以前的说明**（如「此前规定」「变更说明」），便于分析
2. **随时更新设计文档**：功能、策略、阶段状态变更后，同步更新相关文档
3. **术语一致**：统一术语使用，避免混淆
4. **链接检查**：定期检查文档间链接，确保正确
