# 项目价值与定位

> 2026 年视角：P2P-P2P/p2p 项目的核心意义

---

## 一、项目核心定位

**P2P-P2P/p2p** 在尝试做一件目前主流钱包和中心化交易所还没完全做到、或者做得不彻底的事情：

> **让普通人可以用手机浏览器，像用微信/支付宝一样简单地进行"点对点代币交易 + 自动撮合 + 链上结算"，同时保留去中心化的核心属性（无托管、无 KYC、无平台跑路风险）。**

---

## 二、为什么这个方向还有意义？（2026 年视角）

### 2.1 主流钱包的 Swap 是"中心化聚合"

**现状**：
- Trust、MetaMask、Binance Web3 等虽然方便，但本质上是把你引导到 1inch / Uniswap / PancakeSwap 等中心化聚合器或 AMM 池子
- 它们没有真正的"P2P 订单簿撮合"——你挂的限价单本质上是进 AMM 池或被路由器匹配，而不是直接 peer-to-peer
- 手续费最终还是流向 LP、协议或聚合器，而不是你能自定义的分成机制

### 2.2 现有 P2P DEX 的痛点

| 项目 | 优势 | 痛点 |
|------|------|------|
| **Bisq** | 专注 BTC/fiat，去中心化程度高 | 速度慢、流动性差、不支持 ERC-20 通用 Swap |
| **AtomicDEX** | 原子交换好，安全性高 | UI 复杂、链支持有限、没有实时订单簿 |
| **dYdX v4 / Hyperliquid** | 订单簿快，体验好 | 高度中心化（validator / sequencer 控制），不是真正 P2P |
| **主流 DEX** | 流动性高，生态成熟 | 大多数还是 AMM 为主，限价单体验差、滑点大、gas 高 |

### 2.3 本项目试图填补的空白

1. **手机浏览器 + PWA**：门槛极低（无需下载 App、无需桌面），直接"添加到主屏幕"就能用，像用小程序一样
2. **off-chain P2P 撮合 + on-chain 结算**：结合了 Bisq 的 P2P 精神 + dYdX 的订单簿速度 + relayer 代付 gas 的低成本
3. **MatchEngine + libp2p**：节点间直接广播订单、实时匹配，理论上可以做到"零滑点限价单撮合"，而主流 AMM 做不到
4. **自定义分成（FeeDistributor + Governance）**：手续费可以按提案分给开发者/节点运行者/流动性提供者，而不是永远流向协议方
5. **纯手机端架构**：贡献分和核心数据保存在链上，手机钱包即可查看/领取；本地数据用 IndexedDB 缓存，保证离线体验；节点运行由少数 VPS 承担，手机用户无需跑节点（详见 [手机端架构与数据保存策略](./手机端架构与数据保存策略.md)）
6. **桌面版只是可选**：主力手机 PWA，桌面只是锦上添花

## 三、项目最有价值的"差异化意义"

### 3.1 技术差异化

1. **少数几个开源的、手机优先的、真正 P2P 订单簿 DEX**（不是 AMM fork）
2. **libp2p + Electron/PWA + relayer gas 代付**结合得比较完整，这条路很少人走
3. **MatchEngine 稳定运行后**，可以扩展到"手机挂限价单 → 别人手机匹配 → 自动结算"，这比当前钱包的 Swap 更接近"去中心化支付宝"
4. **手续费分成机制 + 开源性质**，有机会吸引小规模开发者/节点运行者，形成社区自驱动

### 3.2 用户体验差异化

- **零门槛**：手机浏览器打开即用，无需下载 App
- **真正 P2P**：订单直接 peer-to-peer 匹配，不经过中心化路由
- **限价单体验好**：off-chain 订单簿，零滑点撮合
- **Gas 成本低**：relayer 代付，用户无需关心 gas

## 四、残酷的现实对比

| 维度 | 主流钱包 Swap<br/>(Trust/MetaMask/Binance) | 本项目（当前状态） | 潜在优势<br/>（如果做成） |
|------|-------------------------------------------|-------------------|-------------------------|
| **使用门槛** | 极低（手机 App 一键） | 低（PWA + WalletConnect） | 相当接近 |
| **流动性** | 高（聚合大池） | 极低（刚起步） | 需要时间积累 |
| **限价单体验** | 差（基本靠 AMM 路由） | 好（off-chain 订单簿） | **明显优势** |
| **Gas 成本** | 中～高（视链） | 低（relayer 代付） | **优势** |
| **真正 P2P 程度** | 低（中心化路由） | 高 | **核心卖点** |
| **手续费去向** | 协议/LP/聚合器 | 可自定义分成 | **长期吸引力** |
| **开发活跃度** | 高（大公司） | 低（目前小项目） | 需要更多贡献者 |

## 五、项目意义总结

**一句话总结**：

> 本项目意义在于：**用最小的门槛（手机浏览器）+ 真正的 P2P 撮合 + 可自定义的分成机制，去挑战主流钱包"方便但中心化"的 Swap 体验。**

---

## 六、当前最缺的（让项目有真正意义）

### 6.1 核心需求

1. **真实流动性**（哪怕先自建小池）
   - 没有流动性，再好的技术也是空谈
   - 建议：先在小范围内（测试网或小规模主网）建立初始流动性

2. **更多测试用户反馈**
   - 需要真实用户的使用体验
   - 建议：在小群或社区内进行内测，收集反馈

3. **持续迭代 MatchEngine 的稳定性与性能**
   - 这是核心竞争优势，必须稳定可靠
   - 建议：持续优化、压力测试、性能监控

4. **一点点社区曝光**（哪怕先在小群里测）
   - 需要让更多人知道这个项目
   - 建议：技术博客、社区分享、开源社区参与

### 6.2 发展路径建议

**短期（1-3 个月）**：
- 完善 MatchEngine 稳定性
- 建立测试网流动性池
- 收集 10-20 个真实用户反馈

**中期（3-6 个月）**：
- 优化 PWA 体验
- 扩大节点网络
- 建立社区治理机制

**长期（6-12 个月）**：
- 主网正式运营
- 形成可持续的流动性
- 建立开发者社区

## 七、项目愿景

**最终目标**：

> 让每个人都能用手机浏览器，像用微信/支付宝一样简单地进行去中心化代币交易，同时享受真正的 P2P 撮合、零滑点限价单、低 gas 成本、可自定义手续费分成等优势。

**核心价值主张**：

- ✅ **去中心化**：无托管、无 KYC、无平台跑路风险
- ✅ **易用性**：手机浏览器即用，无需下载 App
- ✅ **真正 P2P**：订单直接 peer-to-peer 匹配
- ✅ **低成本**：relayer 代付 gas，手续费可自定义分成
- ✅ **开源透明**：代码开源，社区驱动
